# BRIEF
## 現状サマリ（何があるか/何が動くか）
- VSCode 拡張で未コミット時のみ赤いステータス表示を行う
- 通知は出さず、クリックで SCM ビューを開く
- `git status --porcelain` で検知し、一定間隔で更新

## ゴール（最終成果）
- 未コミット状態を見落とさない可視化を VSCode 内で完結させる

## 制約（絶対）
- 通知は出さない（ステータスバーのみ）
- Git がインストール済みであることを前提
- 対象 OS：Win11 中心、将来 Mac

## 作業ルール（承認最小化）
- 作業は「編集→実行→Git」の 3 フェーズに分ける
- 編集フェーズは必要な変更をまとめて適用し、承認は 1 回
- 実行フェーズは実行コマンドを `scripts/run.ps1` にまとめ、1 回の実行で済むようにする（承認は 1 回）
- Git フェーズは commit/push/PR を最後にまとめて行い、途中で細切れにしない
- 追加の承認が必要になりそうなら、先に「これから実行する一覧」を提示してから進める

## 入力/出力
- 入力：VSCode のワークスペース（複数可）
- 出力：ステータスバーの赤いインジケータ

## 主要機能（MVP）
- 未コミット検知（トラッキング/未トラック含む）
- クリックで SCM ビューを開く
- ポーリング間隔の設定

## 不明点（最初に質問すべき点）
- 既定のポーリング間隔
- 未トラックの扱い
